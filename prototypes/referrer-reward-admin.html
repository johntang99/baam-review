Notes / questions for discussion

  • Two-coupon model: friend's code (e.g. FRIEND20) and A's code (e.g. THANKS20) are independent — different code, possibly different discount amount.
  • Redemption method is per-business: some show the page, some say the code at booking, some apply at online checkout — radio picker lets each business choose.
  • Optional: the whole reviewer's reward block can be disabled; in that case the customer just sees the friend share card (current behavior).
  • Tracking (later phase): when A's reward is used, we could log it back to BAAM Studio so the business sees "5 customers redeemed THANKS20 this month."
