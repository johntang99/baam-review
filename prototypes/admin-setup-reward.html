Design notes

  • Universal structure: works for clinics (appointment), stores (no appointment, image of coupon), auto-repair (image of service voucher), home services (image of certificate). Image + markdown text describes any reward type.
  • Image upload: 4MB cap is fine — we auto-compress to ~500KB-1MB on upload for mobile speed. SVGs sanitized server-side to block XSS payloads. Supabase Storage handles the file; the location row stores the URL.
  • Markdown: we'll use a small allow-listed renderer (no raw HTML, no iframes, no scripts) — same approach BAAM Studio uses for testimonial blocks. Bold, italic, lists, links only.
  • Booking checkbox: when unchecked, the booking URL field hides (greyed out) and the customer page omits the "Book now" button entirely — no half-broken CTAs for stores/walk-ins.
  • Trigger rule: reward card only shows after 4★ or 5★ reviews. 1–3★ goes straight to private feedback.
